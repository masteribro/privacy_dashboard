## PART 3: DATABASE - WHERE DATA IS STORED

Your project has 8 tables (think of them as spreadsheets):

### TABLE 1: user
Stores user account information
Columns:
- id (unique identifier)
- username (login name)
- email (contact email)
- password (encrypted)
- created_at (when account was created)

Example data:
| id | username | email | password |
|----|----------|-------|----------|
| 1 | demo | demo@example.com | hashed_password |

### TABLE 2: organisation
Stores company information
Columns:
- id (unique ID)
- name (company name)
- description (what they do)
- logo (emoji icon)

Example data:
| id | name | description | logo |
|----|------|-------------|------|
| 1 | TechFlow Cloud | Cloud storage | ☁️ |
| 2 | ConnectSocial | Social media | 👥 |

### TABLE 3: data_source
Links users to organisations (says "this user's data is with this company")
Columns:
- id
- user_id (which user)
- organisation_id (which company)
- status (active/inactive)
- connected_at (when connected)

### TABLE 4: data_item
Actual personal data (the real sensitive stuff)
Columns:
- id
- data_source_id (which connection)
- category (personal/financial/location/health)
- name (what data? e.g., "email", "phone")
- value (actual value, e.g., "user@example.com")
- purpose (why company has it)

Example:
| id | category | name | value | purpose |
|----|----------|------|-------|---------|
| 1 | personal | Full Name | John Doe | Account identification |
| 2 | financial | Account Type | Premium | Membership status |

### TABLE 5: consent
Stores permissions users have given
Columns:
- id
- user_id (user who gave permission)
- organisation_id (company getting permission)
- purpose (what for? e.g., "Marketing emails")
- consent_type (gdpr or ccpa)
- status (active or withdrawn)
- granted_at (when permission given)
- withdrawn_at (when permission removed)

### TABLE 6: subject_access_request
Stores data export requests (GDPR right to get your data)
Columns:
- id
- user_id (who requested)
- request_date (when requested)
- status (pending/completed/rejected)
- data_format (json or csv)
- completed_at (when data was sent)

### TABLE 7: audit_log
Tracks every action in the system
Columns:
- id
- user_id (who did it)
- action (what did they do? e.g., "view_data", "export_data")
- resource_type (what did they act on? e.g., "consent", "organisation")
- resource_id (which specific one)
- details (extra info)
- ip_address (where they accessed from)
- timestamp (when)
- status (success/failed)

### TABLE 8: user_preference
Stores user settings
Columns:
- id
- user_id
- language (en, es, fr)
- theme (light/dark)
- newsletter (subscribed yes/no)

```python
    if request.method == 'POST':
        username = request.form.get('username')
```

Translation:

ROUTE EXAMPLE 3: Dashboard
```python
@app.route('/dashboard')
@login_required
def dashboard():
    user = current_user
    total_consents = Consent.query.filter_by(user_id=user.id).count()
    active_consents = Consent.query.filter_by(
        user_id=user.id, 
        status='active'
    ).count()
    
    health_score = 70
    if total_consents > 0:
        consent_ratio = active_consents / total_consents
        health_score += int((0.5 - consent_ratio) * 20)
    
    return render_template('dashboard.html', 
                         user=user,
                         health_score=health_score)
```

Translation:
- Line 1: "When user goes to /dashboard..."
- Line 2: "@login_required" means user MUST be logged in
- Line 3: "Get current user"
- Line 4: "Count how many consents user has"
- Line 5-7: "Count how many are active"
- Line 9: "Start with health score of 70"
- Line 10-12: "If user has consents, calculate how many are active"
- Line 13-14: "If ratio is not 0.5 (50%), adjust score"
- Line 16-19: "Send this info to dashboard.html template"

## PART 6: HOW FEATURES WORK

### Feature 1: User Registration
Step by step:
1. User goes to /register
2. See register.html form with username, email, password fields
3. User fills form and clicks "Register"
4. Flask receives form data
5. Code hashes the password (encrypts it)
6. Code creates new User in database
7. User gets logged in
8. Redirected to dashboard

### Feature 2: Viewing Personal Data
Step by step:
1. User logged in, clicks "My Data"
2. Goes to /mydata route
3. Python queries database:
   - Find all data_source records for this user
   - For each data_source, find all data_item records
   - Find organisation info
4. Organize data by category
5. Pass to mydata.html template
6. Template loops through and displays in tables

### Feature 3: Withdrawing Consent
Step by step:
1. User goes to Consents page
2. See all consents they've given
3. Click "Withdraw" button on a consent
4. Goes to /withdraw-consent route
5. Python updates consent status from 'active' to 'withdrawn'
6. Records when it was withdrawn
7. Logs action in audit_log table
8. Redirected back to consents page with success message

### Feature 4: Exporting Data
Step by step:
1. User clicks "Export My Data" button
2. Goes to /export-data route
3. Python queries database for ALL user data
4. Converts it to JSON format
5. Logs the export in audit_log
6. Sends file to user's download folder
7. File name: my-privacy-data.json

### Feature 5: Privacy Health Score
Calculation:
```
Base Score = 70

IF user has consents:
    Calculate what percentage are active
    IF less than 50% active = good = score goes UP
    IF more than 50% active = risky = score goes DOWN

IF user connected to more than 5 organisations:
    Penalty: -2 points per extra organisation

Final score capped between 0-100
```

Example:
- User has 3 consents, all active (100% active)
- Score = 70 + int((0.5 - 1.0) * 20) = 70 - 10 = 60
- User withdraws 1 consent (66% active)
- Score = 70 + int((0.5 - 0.66) * 20) = 70 - 3 = 67

## PART 8: COMPLETE WALKTHROUGH - USER LOGS IN

Here's what happens step-by-step when user logs in:

### STEP 1: User Visits Login Page
- User types: http://localhost:5001/login
- Browser sends GET request to Flask
- Flask runs the login() function
- Function returns login.html template
- Browser shows login form

### STEP 2: User Fills Form
- User types username: "demo"
- User types password: "demo123"
- User clicks "Login" button

### STEP 3: Form Submits (POST Request)
- Browser sends POST request to /login with form data
- Flask receives the POST request
- Flask runs login() function again (now with POST data)

### STEP 4: Python Processes Login
```python
@app.route('/login', methods=['GET', 'POST'])
def login():
    if request.method == 'POST':
        # Get form data
        username = request.form.get('username')  # Gets "demo"
        password = request.form.get('password')  # Gets "demo123"
        
        # Query database for user with this username
        user = User.query.filter_by(username=username).first()
        # Finds user with id=1 from database
        
        # Check if user exists AND password is correct
        if user and check_password_hash(user.password, password):
            # Password is correct!
            
            # Log the user in
            login_user(user)
            
            # Log this action
            log_audit("login", "user", user.id, "User logged in")
            
            # Send user to dashboard
            return redirect(url_for('dashboard'))
        else:
            # Password is wrong
            flash('Invalid credentials', 'danger')
            return render_template('login.html')
```

### STEP 5: User Logged In - Redirect to Dashboard
- Flask sends redirect to /dashboard
- Browser automatically goes to /dashboard

### STEP 6: Dashboard Page Loads
```python
@app.route('/dashboard')
@login_required
def dashboard():
    user = current_user  # Gets logged-in user (demo)
    
    # Count consents
    total_consents = Consent.query.filter_by(user_id=user.id).count()

    # Queries database: SELECT COUNT(*) FROM consent 
    # WHERE user_id=1 AND status='active'
    # Returns: 3
    
    # Calculate health score
    health_score = 70
    if total_consents > 0:
        consent_ratio = active_consents / total_consents  # 3/3 = 1.0
        health_score += int((0.5 - consent_ratio) * 20)  # 70 + (0.5-1.0)*20 = 60
    
    # Get organisations
    data_sources = DataSource.query.filter_by(user_id=user.id).all()
    # Queries: SELECT * FROM data_source WHERE user_id=1
```

## PART 9: ERROR MESSAGES & DEBUGGING

### Common Errors:

**Error: "no such column: consent.consent_type"**
Means: Database table doesn't have that column
Fix: Run database migration or reset database

**Error: "Could not build url for endpoint 'logout'"**
Means: The logout() function doesn't exist in app.py
Fix: Make sure logout route is defined

**Error: "'None' has no attribute 'strftime'"**
Means: Variable is empty/None, trying to format as date
Fix: Check if value exists before calling method

### How to Debug:

Add print statements:
```python
def dashboard():
    user = current_user
    print(f"User: {user.username}")  # See if user loaded
    print(f"User ID: {user.id}")
    
    consents = Consent.query.filter_by(user_id=user.id).all()
    print(f"Consents: {consents}")  # See if query worked
    print(f"Count: {len(consents)}")
```

Then run and check terminal output.
